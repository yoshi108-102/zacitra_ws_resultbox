infra/ 構成説明

このディレクトリは Terraform で AWS 上に 「静的サイト配信基盤 + Cognito 認証基盤」を構築するための定義です。

1. 何を作る構成か

  • フロントエンド配信: S3 + CloudFront（OAC で保護）
  • 認証: Cognito User Pool + Hosted UI
  • SPA 向けのルーティング対応: CloudFront で 403/404 -> /index.html

目的は、公開サイトを安全に配信しつつ Cognito でログイン連携できる最小構成を作ることです。

2. ファイルごとの役割

versions.tf

Terraform と AWS Provider のバージョン制約を定義。

providers.tf

AWS Provider のリージョン設定（var.aws_region）。

variables.tf

入力変数（project_name など）を定義。

main.tf

主要リソース（S3, CloudFront, Cognito）を作成。

outputs.tf

デプロイ後に参照する値（ドメイン名、ID）を出力。

terraform.tfvars.example

環境ごとの変数設定サンプル。

3. main.tf のリソース詳細

  • aws_s3_bucket.site: サイトアセット配置用バケット
  • aws_s3_bucket_public_access_block.site: 直接公開を禁止
  • aws_s3_bucket_versioning.site: バージョニング有効化
  • aws_s3_bucket_server_side_encryption_configuration.site: SSE-AES256
  • aws_cloudfront_origin_access_control.site: CloudFront から S3 への署名アクセス
  • aws_cloudfront_distribution.site: HTTPS 配信・キャッシュ・SPA エラーハンドリング
  • aws_s3_bucket_policy.site: CloudFront のみ s3:GetObject を許可
  • aws_cognito_user_pool.main: メールログインを前提にしたユーザープール
  • aws_cognito_user_pool_client.site: OAuth code flow 用アプリクライアント
  • aws_cognito_user_pool_domain.main: Hosted UI ドメイン作成

4. 入力値と出力値

主な入力値（terraform.tfvars で指定）

  • project_name: リソース命名の接頭辞
  • aws_region: デプロイリージョン（例: ap-northeast-1）
  • cognito_domain_prefix: Cognito Hosted UI の一意ドメイン接頭辞
  • tags: 共通タグ

主な出力値（terraform output で取得）

  • cloudfront_domain_name
  • site_bucket_name
  • cognito_user_pool_id
  • cognito_user_pool_client_id
  • cognito_hosted_ui_domain

5. 運用上の注意

  • terraform.tfstate には環境固有の情報が含まれるため取り扱い注意
  • 本番運用ではリモートステート（例: S3 + DynamoDB Lock）を推奨
  • .terraform/ は作業ディレクトリのため通常は管理対象外
